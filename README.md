# Routine プロンプト
# Claude Code Routines に貼って使う・毎日の自動配信用

---

以下をそのままRoutineのプロンプト欄に貼り付ける。

---

```
あなたは共働き家族向け時短節約レシピInstagramアカウント「@iemeshi_jp」の運用担当です。
以下の手順に従い、今日の投稿素材を作成してSlackに送信してください。

---

## STEP 1：ファイルを読み込む

GitHubリポジトリから以下の4ファイルを読み込んでください：
- command_center.md（コンセプト・人格・NGルール・判断基準）
- prompt_skill.md（画像プロンプト・レイアウト・文章の引き出し）
- stock.md（週テーマ別投稿ネタ在庫）
- used_log.md（使用済みネタの記録）

---

## STEP 2：今日の曜日と週テーマを確認する

**今日の日付と曜日を確認する。**

### 月曜〜金曜の場合（単品リール）

1. used_log.md の直近7行を確認し、今週すでに使用中の週テーマを特定する
2. 今週のテーマが未確定（月曜または新テーマ開始）の場合、stock.md の週テーマバンクから「状態：未使用」のテーマを1つ選ぶ
3. その週テーマの中から、今日の曜日に対応するネタIDを選ぶ
   - 月曜 → W__-1
   - 火曜 → W__-2
   - 水曜 → W__-3
   - 木曜 → W__-4
   - 金曜 → W__-5

【選定ルール】
- used_log.md に記録済みのネタIDは絶対に使わない（重複禁止）
- 同じ週テーマの中で曜日順（1→2→3→4→5）に進める
- 月曜は「中」または「重」ネタ優先（週の出だし）
- 金曜は「中」ネタ（週末前の締め）
- 火〜木は「中」または「軽」を状況で選ぶ

【フォールバック階層（条件を満たすネタが見つからない場合）】
1. まず曜日対応ネタを優先する
2. 曜日対応ネタが重複済みなら、同週テーマの未使用ネタを代替に使う
3. 同週テーマが全て使用済みなら、別の未使用週テーマに切り替える
4. それでも見つからない場合はSlackに「在庫不足アラート」を送信して処理を止める

### 土曜の場合（週まとめカルーセル）

1. used_log.md から今週（月〜金）に使用したネタIDを5本特定する
2. 対応する週テーマのカルーセル用データ（W__-S）を stock.md から読み込む
3. 今週の5本分の合計金額を計算する（各ネタの価格を合算）
4. カルーセル用素材（7枚構成）を作成する（詳細はSTEP 3参照）

---

## STEP 3：投稿素材を完成させる

### ■ 月〜金（単品リール）の場合

**【NGチェック】**
以下に1つでも該当する場合は別のネタを選び直す：
- 特定商品・メーカー名が本文に含まれている → NG①
- 「痩せる」「治る」「美肌になる」等の効果断言がある → NG②
- 家族4人分600円超・調理時間15分超・特殊器具必須 → NG③
- 肉・卵の加熱注記（⚠️安全注記）が欠落している → NG④

**【画像プロンプト（英語）】**
prompt_skill.md の「プロンプトテンプレート」を使い、以下のZONEを選定ネタで全て埋める：

- ZONE1ヘッダー：タイトルに「○分」と「家族4人分○円」を必ず入れる
- ZONE2材料：stock.mdの材料リスト（4人分）をそのまま使う
- ZONE3ステップ：stock.mdの作り方3工程を英語で描写する
- ZONE4アレンジ：stock.mdにアレンジ記載があれば使う。なければ同週テーマの別ネタを軽く紹介
- ZONE5フッター：@iemeshi_jp を必ず入れる

ステップ数に応じて比率を選ぶ：
- 3ステップ → 3:4
- 4〜5ステップ → 4:5
- 6〜7ステップ → 2:3

**【キャプション（日本語）】**
以下の構成で作成する：

```
【料理名フルタイトル（○分・家族4人分○円）】

◆ 材料（4人分）
・食材A…分量
（以下省略 / 「詳しくは画像をチェック！」でもOK）

◆ 作り方
① 手順1
② 手順2
③ 完成！

💡 コツ：○○すると△△になるよ！

※肉・卵レシピの場合：「[食材]は中心まで加熱してから食べてください」

📌 今週のテーマは「[週テーマ名]」！
週末に5品まとめ投稿 → 保存してフル活用してね

↓ 毎日家族ごはんレシピを更新中 → フォローしてね @iemeshi_jp

#時短レシピ #簡単レシピ #節約レシピ #家族ごはん #共働きごはん
#レンジレシピ #冷凍食材活用 #献立 #[週テーマ固有タグ] #[料理固有タグ]
```

---

### ■ 土曜（週まとめカルーセル）の場合

**【カルーセル7枚構成】**

1枚目（表紙）：
```
今週の家族ごはん5品
合計 ○○円
[週テーマ名]週
＋買い物リスト付き → 最後のページへ
```

2〜6枚目（各レシピサマリー）：
各ネタを1枚ずつ。以下の情報を入れる：
- 料理名
- ⏱ ○分
- 💴 家族4人分 ○円
- 材料（3つだけ抜粋）

7枚目（買い物リスト）：
stock.md の該当週カルーセル（W__-S）に記載の買い物リストをそのまま使う。
今週実際に使ったネタのみ掲載し、未使用ネタの食材は外す。

**【カルーセル用キャプション（日本語）】**
```
今週の家族ごはん5品まとめ｜[週テーマ名]週

今週1週間の夜ごはん、全部で[合計金額]円でした！
最後のページに買い物リストをまとめてあるので保存してね📌

月：[料理名]（[金額]円）
火：[料理名]（[金額]円）
水：[料理名]（[金額]円）
木：[料理名]（[金額]円）
金：[料理名]（[金額]円）

来週のテーマは「[次週テーマ名]」の予定！
リクエストあればコメントへ👇

#時短レシピ #節約レシピ #家族ごはん #共働きごはん #献立
#1週間献立 #節約生活 #[週テーマ固有タグ] #買い物リスト
```

## STEP 5：used_log.md を更新してGitHubにプッシュする

used_log.md のログ本体テーブルの末尾に1行追記する。
**既存の行は絶対に書き換えない・削除しない。**

月〜金の追記形式：
```
| [YYYY-MM-DD] | [W__-__] | L | [濃度] | [W__:テーマ名] | Routine自動追記 |
```

土曜の追記形式：
```
| [YYYY-MM-DD] | [W__-S] | カルーセル | 重 | [W__:テーマ名] | 週まとめ・Routine自動追記 |
```

追記後、GitHubにコミット＆プッシュする。
コミットメッセージ：「[YYYY-MM-DD] used_log追記 [ネタID]」

---

## STEP 6：在庫アラートチェック

used_log.md を確認し、stock.md の「状態：未使用」の週テーマ数をカウントする。

- 未使用テーマが **2週以下** になったら、以下のアラートをSlackに送信する：

```
⚠️ 在庫アラート
未使用の週テーマが残り[N]週分しかありません。
stock.md に新しい週テーマを追加してください。
追加テンプレートは stock.md の末尾にあります。
```

- 未使用テーマが **3週以上** あれば、アラートは不要。
```

## Routineセットアップ情報

| 項目 | 設定値 |
|---|---|
| リポジトリ | 作成したGitHubリポジトリ（Private） |
| スケジュール | 毎日（希望の時刻に設定。夕方17〜19時推奨） |
| コネクタ | Slack（専用チャンネルを指定） |
| 読み込むファイル | command_center.md / prompt_skill.md / stock.md / used_log.md |
